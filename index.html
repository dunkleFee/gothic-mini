1.2 Главный цикл и стейт-машина

Вся игровая логика проходит через один объект Game. Переключение состояний — через StateMachine с push/pop стеком, чтобы диалог «накладывался» поверх мира без разрушения его состояния.
# Псевдокод главного класса class Game: states = { 'WORLD': WorldState, # основной геймплей на карте 'DIALOG': DialogState, # диалог с NPC (поверх мира) 'COMBAT': CombatState, # бой на карте (реальное время) 'LOOT': LootState, # окно лута контейнера 'MAP': MapState, # экран мировой карты 'MENU': MenuState, # пауза / главное меню 'INTERIOR':InteriorState, # внутри здания (суб-карта) } stack: List[State] # стек состояний (push/pop) def run(self): while running: dt = clock.tick(FPS) / 1000 events = pygame.event.get() self.handle_events(events, dt) self.update(dt) self.draw() pygame.display.flip() def handle_events(self, events, dt): input_cmds = self.input_manager.process(events) self.stack[-1].handle(input_cmds, dt) def update(self, dt): for state in self.stack: state.update(dt) # нижние состояния «живут», но не принимают ввод def draw(self): for state in self.stack: state.draw(self.screen)

class TileRenderer: LAYERS = ['GROUND', 'OBJECTS', 'SHADOWS', 'ENTITIES', 'EFFECTS'] def draw_tile(self, surface, tile, col, row): # 1. Вычислить экранную позицию с учётом камеры и 2.5D-смещения sx = (col - cam.x) * TILE_W sy = (row - cam.y) * TILE_H + tile.height_offset # вертикальный сдвиг # 2. Глубина → яркость (дальние объекты темнее) depth_factor = 1.0 - (row - cam.y) * 0.02 color = lerp_color(tile.base_color, DIM_COLOR, depth_factor) # 3. Рисуем символ тайла surface.blit(font.render(tile.char, color), (sx, sy)) # 4. Тень под высокими объектами (стены, деревья) if tile.is_tall: draw_shadow(surface, sx, sy + SHADOW_OFFSET, alpha=60) def draw_frame(self, surface, zone, cam): for layer in self.LAYERS: # Сортировка объектов по row для правильного z-order tiles = zone.get_layer(layer) for tile in sorted(tiles, key=lambda t: t.row): self.draw_tile(surface, tile, t.col, t.row)

1.5 World Graph и переходы между зонами

# world_graph.json — пример фрагмента { "zones": [ { "id": "BARRIER_START", "name": "У Барьера", "map": "zone_start.txt", "danger": 1, "spawn_table": "spawn_weak" }, { "id": "ROAD_N1", "name": "Северная тропа", "map": "zone_road_n1.txt", "danger": 2, "spawn_table": "spawn_medium" }, { "id": "OLD_CAMP", "name": "Старый лагерь", "map": "zone_old_camp.txt", "danger": 0, "spawn_table": "none" } ], "edges": [ { "from": "BARRIER_START", "to": "ROAD_N1", "exit": "north", "condition": "none" }, { "from": "ROAD_N1", "to": "OLD_CAMP", "exit": "north", "condition": "none" }, { "from": "OLD_CAMP", "to": "SWAMP_PATH", "exit": "west", "condition": "rep_old >= 10" } ] }

Логика смены зоны

  1. Игрок выходит за границу текущей карты (тайл-триггер EXIT_{DIR})
  2. WorldGraph.get_transition(current_zone, direction) → находит ребро
  3. Проверяется condition (репутация, квест, уровень)
  4. Если OK → ZoneManager.load_zone(next_id), игрок помещается на противоположный край
  5. При входе в дверь (тайл DOOR) → StateMachine.replace(INTERIOR, interior_id)

1.6 Монстры, бой и лут

class Monster(Entity): hp: int; max_hp: int; damage: int; speed: float state: Enum # PATROL | CHASE | ATTACK | DEAD patrol_path: List[Point] aggro_range: int loot_table: str # ссылка на таблицу дропа def update(self, dt, player_pos): if self.state == PATROL: self.follow_patrol(dt) if dist(self.pos, player_pos) < self.aggro_range: self.state = CHASE elif self.state == CHASE: self.move_towards(player_pos, dt) if dist(self.pos, player_pos) < ATTACK_RANGE: self.state = ATTACK elif self.state == ATTACK: self.attack_timer -= dt if self.attack_timer <= 0: self.do_attack(player); self.attack_timer = self.attack_cd class Container(Entity): state: Enum # CLOSED | OPEN | LOOTED loot_table: str char_by_state = {CLOSED: '▣', OPEN: '▢', LOOTED: '□'} def interact(self, player): if self.state == CLOSED: loot = LootTable.roll(self.loot_table) self.state = OPEN return LootEvent(loot) # → открывает LootWindow

1.7 Web-сборка и Telegram Mini App

Рекомендуемый инструмент сборки: pygbag (pip install pygbag). Он упаковывает Pygame в WASM через Emscripten, создаёт готовую папку web/ с index.html.

Этапы сборки

  1. Подготовить код под asyncio: заменить синхронный game loop на async def main() + asyncio.sleep(0)
  2. Запустить pygbag --build main.py → получить папку build/web/
  3. Проверить локально: pygbag main.py (localhost:8000)
  4. Залить содержимое build/web/ на GitHub Pages (или любой статик-хостинг)
  5. В BotFather → /newapp или /setmenubutton → указать URL страницы

4.1 Чек-листы логики локаций и квестов

Переходы и интерьеры

CRITICAL
При выходе за край карты: игрок появляется на правильном краю следующей зоны (не в стене, не в воде, не на монстре)
CRITICAL
Вход/выход из интерьера сохраняет позицию на внешней карте (игрок не телепортируется в (0,0) после выхода из дома)
WARNING
Переход с условием (rep_old ≥ 10): при недостаточной репутации — сообщение от стражника, а не молчаливый softlock
NORMAL
Все зоны имеют триггеры выхода со всех 4 сторон, или явную визуальную блокировку (стена Барьера, горы) на непроходимых сторонах
NORMAL
Туннели/порталы двунаправленны: из A→B и из B→A ведут на правильные позиции

Квесты и состояния

CRITICAL
Нет «зависших» квестов: завершение квеста всегда переводит его в DONE, никогда не остаётся в ACTIVE без цели
CRITICAL
Награда за квест выдаётся ровно один раз (защита от дублирования при повторном открытии диалога)
WARNING
После завершения квеста NPC переключаются на «завершённый» диалог, а не повторяют исходное предложение
WARNING
Ветвящиеся квесты (Q01-3 «Слежка»): оба пути имеют работающие последствия, ни один не ведёт в тупик
NORMAL
Репутация корректно меняется при всех событиях (диалоги, квесты, атака NPC) и влияет на доступность переходов

4.2 Проверки боя, монстров и лута

Спавн монстров

CRITICAL
Монстры не спавнятся на клетке игрока, на тайлах WALL/WATER, на тайлах дверей и переходов
CRITICAL
Монстры не блокируют навсегда тайлы перехода между зонами (выходы должны оставаться проходимыми)
WARNING
Стая падальщиков (2–4 особи): все особи не стакаются на одном тайле, патрулируют разными путями

Поведение и баланс

NORMAL
Монстр в состоянии PATROL не атакует игрока вне aggro_range; в состоянии CHASE преследует до потери из виду
NORMAL
Смерть монстра: тело заменяется символом «%», через N секунд исчезает. Лут выпадает на пол (тайл) или автособирается (настройка)
WARNING
Стартовые зоны (danger=1): монстры слабы настолько, чтобы игрок с начальным ножом мог убить их за 3–5 ударов без критических потерь HP

Контейнеры и лут

CRITICAL
Контейнер в состоянии LOOTED не возвращает лут при повторном взаимодействии. Состояние сохраняется после смены зоны и перезагрузки
WARNING
LootWindow: кнопка «Взять всё» корректно добавляет все предметы в инвентарь, не дублирует, не теряет стекируемые предметы
NORMAL
Квестовые предметы (письма, ключи) всегда присутствуют в соответствующем контейнере независимо от броска таблицы лута

4.3 Telegram Mini App — специфичные тесты

Отображение и ориентация

CRITICAL
Android (Chrome WebView): нет горизонтальной полосы прокрутки, canvas не обрезан, ASCII-символы не замещаются □
CRITICAL
iOS (Safari WebView): нет резинового скролла, viewport зафиксирован (meta viewport + touch-action:none на canvas)
WARNING
Поворот экрана: игра корректно перестраивает layout через 300–500 мс после смены ориентации, без перезагрузки
NORMAL
Размер шрифта ASCII: минимум 11px на 375px ширине. Символы различимы без зума.

Управление (тач)

CRITICAL
Виртуальный джойстик реагирует на первый тач с задержкой < 50мс. Нет «мертвой зоны» больше 15% радиуса
CRITICAL
Мультитач: одновременное движение (джойстик) и действие (кнопка атаки) не конфликтуют
WARNING
В диалоговых окнах тап по варианту ответа не проходит «насквозь» на игровую карту под ним

Telegram WebApp API

WARNING
WebApp.expand() вызван до начала игры; игра занимает 100% viewport без лишних отступов браузера
WARNING
Сохранение через CloudStorage.setItem: обработан callback с ошибкой (quota exceeded / API недоступен) — локальный fallback через localStorage
NORMAL
Нет нативных alert/confirm/prompt браузера — всё через внутриигровой UI (они блокируют WebView в Telegram)
NORMAL
Desktop Telegram (WebApp в десктопном клиенте): клавиатурный ввод перехватывается игрой, не уходит в поле чата Telegram

Производительность

WARNING
FPS ≥ 30 на среднем Android (Snapdragon 665+) при полной карте + 10 монстров + эффекты теней
NORMAL
Размер WASM + ассетов ≤ 10 МБ для комфортной загрузки в мобильном WebView без прелоадера
